# Priority Mapping Guide

When creating tasks, suggest appropriate priorities based on impact and urgency.

## Priority Levels

- **P0 (Critical)**: Production down, data loss, security vulnerability
- **P1 (High)**: Major feature broken, blocks other work, high-impact bug
- **P2 (Medium)**: Standard feature work, moderate bugs, improvements
- **P3 (Low)**: Minor bugs, small enhancements, nice-to-haves
- **P4 (Backlog)**: Future work, ideas, technical debt

## Usage Guidelines

- Default to **P2** for standard feature work
- Escalate to **P1** if the issue blocks other work or affects core functionality
- Reserve **P0** for true emergencies (site down, data loss, security breach)
- Use **P3** for minor cosmetic issues or small enhancements
- Use **P4** for ideas and future considerations that aren't actively planned
