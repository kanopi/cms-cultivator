Based on my research, here's what I found:

Current Situation

Good news: Both projects are GPL-2.0-or-later licensed, so they're compatible!

Challenge: Claude Code does not currently support plugin dependencies. There's an
https://github.com/anthropics/claude-code/issues/9444 for this, but it's not yet implemented.

Integration Options

Option 1: Setup Command (Recommended)

Create a /setup-wordpress-skills command that automates the installation:

/setup-wordpress-skills

This would:
1. Clone the WordPress agent-skills repository to a temp location
2. Run their build script (node shared/scripts/skillpack-build.mjs --clean)
3. Run their install script (node shared/scripts/skillpack-install.mjs --global)
4. Skills get installed to ~/.claude/skills/ where Claude Code discovers them
5. Report which WordPress skills are now available

Benefits:
- One command to set everything up
- WordPress skills stay updated via their repo
- Clear separation: CMS Cultivator plugin skills + WordPress global skills
- Users can update WordPress skills independently

Option 2: Git Submodule + Custom Installer

Add WordPress agent-skills as a git submodule and create our own install script:

git submodule add https://github.com/WordPress/agent-skills.git vendor/wordpress-agent-skills

Then create /install-wordpress-skills that runs the build/install from the submodule.

Benefits:
- Version pinning (control which version gets installed)
- Included in plugin repo

Drawbacks:
- More maintenance burden
- Submodules can be tricky for users

Option 3: Documentation Only

Simply document in our README/docs that users can optionally install WordPress agent-skills:

## Optional: WordPress Agent Skills

For enhanced WordPress development, install the official WordPress agent-skills:

\`\`\`bash
git clone https://github.com/WordPress/agent-skills.git
cd agent-skills
node shared/scripts/skillpack-build.mjs --clean
node shared/scripts/skillpack-install.mjs --global
\`\`\`

These skills work alongside CMS Cultivator and provide deep WordPress expertise.

Benefits:
- Zero maintenance
- Users choose if they want it
- No complexity added to our plugin

Drawbacks:
- Manual process for users
- Skills might get out of sync

My Recommendation

Go with Option 1 (Setup Command). Here's why:

1. User-friendly: One command installs everything
2. Maintains separation: WordPress skills go to ~/.claude/skills/, ours stay in plugin
3. Stays current: Users pull latest WordPress skills anytime
4. Optional: Users can skip if they don't need WordPress-specific features
5. Compatible: When Claude Code adds dependency support, we can migrate easily

- Progress reporting
- Error handling
- Verification that skills were installed successfully
- List of available WordPress skills after installation
